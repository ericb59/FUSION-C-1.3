bin2froh - Converts binary file(s) to/from C header file(s)

Synopsis

bintoh [options] [file...]
binfromh [options] [file...]

Description

Binary files, like Bitmap, have binary format ,while C compiler requires that they are converted to char arrays in a C header file. bintoh converts binary file(s) TO C header file(s), while binfromh converts binary file(s) FROM C header file(s).

The programs accept multiple filenames and wildcards as their arguments. You may also use them in a pipe. If either program finds its input redirected, it will process stdin and place the output on stdout.

Options

-a
Abort processing files on error in any file in a list of files. Normally, the program will simply skip to process the next file on the command line when it encounters any errors. This option causes it to abort on errors.
-c <num>
<num> of columns of the C-style char array.Note this option is only applied when binary file(s) are converting to C header files.
-e <name>
Uses a specific output file name extension, with or without a leading period (.). the program inserts the period automatically if it does not appear in <name>. If no extension is given, the default extension is "h" for C header files, and "bin" for binary files. Note the program would only change the extension of the corresponding input file, while keep all others the same.
-f
Convert binary file(s) FROM C header file(s). This forces the program to convert the file in a particular direction. By default, if the program is named bintoh or bin2h it will assume that the input file is in binary format and convert it to C format. If the program is named binfromh it will assume that the input file is in C format and convert it to binary format. Using the -f option forces the program to convert binary file(s) FROM input file(s) regardless of how the program is named. Likewise, using the -t option forces the program to convert input file(s) TO C header file(s) regardless of the name of the program.
-h
Display a short help screen on the program usage and quit.
-l <file>
Log debug messages to <file>. Note that if your command line has an error, such as when you specify an unknown option, the error message for the command line option error will be issued to stderr instead and not logged.
-o
Overwrite output file(s) if already exist (no backup). By default, the program will create a backup file if the output file already exist. Use -o will not create the backup file.
-s <name>
Symbol <name> of the C-style char array. Note this option is only applied when binary file(s) are converting to C header files. And it cannot separate the symbol names for multiple input files. The default symbol name is Bin_Raw.
-t
Convert binary file(s) to C header file(s). See the -f option above for more information.
-v
Verbose.
-V
Show version message and quit.
